# 以下是基于上述内容的 Cursor 最佳实践思维导图

```
Cursor 最佳实践
│
├── 1. 提示词 (Prompting)
│   ├── 写具体、可验证的提示
│   │   ├── 模糊 → "修复这个 bug"
│   │   └── 具体 → "登录表单点击按钮时会提交两次，请找到原因并修复"
│   ├── 使用 @ 引用上下文
│   │   ├── 知道文件 → @文件名 指定
│   │   ├── 不确定 → 无需标注，Agent 自动搜索
│   │   └── 引入无关文件会干扰 Agent 判断
│   └── 使用 @Past Chats 引用历史对话
│
├── 2. 规划 (Planning)
│   ├── 触发方式：Shift+Tab 切换 Plan Mode
│   └── 执行流程
│       ├── Step 1 搜索代码库相关文件
│       ├── Step 2 提出澄清问题
│       ├── Step 3 生成详细实施计划
│       └── Step 4 等待确认后再编码
│           └── 保存计划到 .cursor/plans/（团队共享）
│
├── 3. 规则 (Rules)
│   ├── 规则类型
│   │   ├── Always Apply       → 全局规范，每次对话应用
│   │   ├── Apply Intelligently → Agent 根据描述判断
│   │   ├── Apply to Specific Files → 匹配路径模式时应用
│   │   └── Apply Manually     → 手动 @mention 触发
│   ├── 好的规则原则
│   │   ├── 保持 500 行以内
│   │   ├── 聚焦、可操作，含具体示例
│   │   ├── 引用文件而非复制内容
│   │   └── 只在 Agent 反复犯同一错误时才添加
│   ├── 避免在规则中
│   │   ├── 复制整个风格指南（用 linter 代替）
│   │   ├── 记录 npm/git 等常识性命令
│   │   └── 添加极少触发的边缘情况
│   └── 将规则纳入 git 版本控制
│
├── 4. 对话管理 (Context Management)
│   ├── 开启新对话的时机
│   │   ├── 切换到不同任务或功能
│   │   ├── Agent 持续出错或陷入循环
│   │   └── 完成了一个完整的工作单元
│   └── 继续当前对话的时机
│       ├── 在同一功能上迭代
│       ├── Agent 需要之前的上下文
│       └── 调试它刚刚构建的内容
│
├── 5. 审查与迭代 (Review & Iterate)
│   ├── 使用 diff 视图实时观察改动
│   ├── 发现偏差 → Escape 中断并重新引导
│   ├── 完成后 Review → Find Issues 专项检查
│   └── AI 代码可能看似正确但存在隐患，务必认真审阅
│
└── 6. 可验证目标 (Verifiable Goals)
    ├── 使用类型语言 → 类型检查器即时反馈
    ├── 先写测试 → Agent 迭代直至通过
    ├── 配置 linter → 自动捕获质量问题
    └── 推荐工作流
        ├── Step 1 Agent 先写测试
        ├── Step 2 确认测试失败
        └── Step 3 Agent 实现代码使测试通过
```

---

​**核心逻辑主线**​：

```
明确目标 (Plan) → 给定上下文 (Context) → 精准提示 (Prompt)
       → 规则约束 (Rules) → 实时审查 (Review) → 可验证迭代 (Iterate)
```

这六个模块相互支撑：规划减少返工，规则保持一致性，上下文管理保持 Agent 专注，可验证目标让迭代有据可依，审查则是整个流程的质量保障。
